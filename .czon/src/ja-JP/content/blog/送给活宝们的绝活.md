2017年2月5日 追記：

先日、たまたま集まりで大学も同じだった先輩に会い、雑談の中で高校はどこだったか聞かれ、「黄河です」と答えました。彼は驚いて、「じゃあ、君は学校のトップだったに違いないね！」と言いました。

大学に入ってから、この質問は西安出身の同級生から何度もされ、返事は上記のような驚きか、あるいはその高校を知らないというものでした。

実は、むしろ「私は高校でビリでした」と言えたらいいのにと思います。

<!--more-->

## タイトルについての説明

タイトルには、二つのとても口語的な言葉が含まれています：**お調子者**と**極意**。分かる人には分かるとは思いますが、説明する責任もあると考え、一応説明しておきます。

この二つの言葉には、間違いなく私の高校時代の担任——曹先生（老曹）の口癖という、とても濃い個人的な色彩が帯びています。曹先生の教え子として、このような言葉は三年間の長い時間の中で何度聞いたか分かりませんが、卒業して三年たった今でも、私はまだ一人前になったとは言えず、ただ門の前に立ち、首を突っ込んで中を覗き見た程度に過ぎないと言えるでしょう。**どんな門なのか、門とは何なのか**については、よく考えてみてほしいと思います。

この一篇を、高校からの諸君、後輩たちに贈ります。

## 自己紹介

ある種の、完全には身についていない職業的習性から、この文章の読者をあまり限定したくないため、後輩たち以上に制限的な仮定を置くこともありません——私は君たちより三歳年上の先輩であるべきでも、六歳年上の先輩であるべきでもないのです。
そのような考えから、君には単に私を、私がこれから語るような**基本的な**、**余計なレッテルを一切貼られていない**ただの**人間**として見てほしい——私がこの文章を書く目的を、君にはっきりと理解してほしいのです。

## 司馬昭の心

なぜこの文章を書くのか、私は長い間悩みましたが、人に贈る言葉として正当な理由をなかなか見つけられませんでした。私たちはこんなにも矛盾しています：いったいどれほどの力量があれば、ある高みに立って人を教える資格があるというのでしょうか？**その人は、その資格を持つ必要もなければ、その資格を持つ程度に達することも永遠に不可能なのです。**
それならば、いっそのことこの資格の問題は考えず、書くのは、自分自身の心の中に長く潜んでいた、語りたいという衝動があるからです。人生の経験を分かち合いたいのです。しかし、私の本心は、君たちに語りかけることで、（かつての）自分自身に対して何らかの総括をし、けじめをつけたいという思いです：互いに利益があれば、それも悪くないでしょう。

## 三人行えば必ず我が師あり

高校時代、私は自分が非常に頭が良い人間だと思い込んでいたので、今に至るまで、誰かがどれだけ頭が良いからといって、その人を尊敬したことはありません。それでも、頭の良い人と付き合うのは好きでした。理由は、コミュニケーションが楽だからです。**しかし、当時の私は、頭の良さと若さの違いを理解していませんでした。**

高校一年の時に、Jと呼んでいた同級生がいました。
Jの頭はとても回転が速く、当時の私も彼が賢いと思っていました。光るものがあれば、お互いに惹かれ合うものです。高校の教科書の知識は偏っていて、穴だらけ（知識の穴）でした。特に物理、化学、生物の三科目は、ほぼどのページをめくっても、教科書がほのめかしているのに解説していない問題が見つかるほどでした。私たちは毎日そのような問題を抱えて、隣の職員室に先生を煩わせに行きました——これはとても称賛に値する学習態度です——時には先生が少し考えて答えを教えてくれ、時には`「受験に出ないから」`という理由でごまかされました。今思えば、私たちの質問が確かにひねくれすぎていたのかもしれませんが、当時私たちはすぐにそれを`「中国の教育制度はゴミを製造する工場だ」`という高みにまで引き上げてしまいました。本当に賢かったのです。
すぐに私たちは調子に乗り始め、Jはさらに一歩進みました。おそらく本当にそのような理由からでしょう、今Jのことを思い返すと、彼の口から三度に一度は`「知能指数」`という言葉が出て、そして`「バカばっかり。」`という満ち足りた傲慢さに満ちた一言だけが残っています。

**これが、思うだけで学ばないことです**

確かに、私はよく星や海についてのことを考え、夜の12時まで考え続けていました。心の中には太陽と月がありましたが、`「駿馬も一躍して十歩は跳べず、駑馬も十駕すればその功は捨てずにいることにある」`という道理が分かっていませんでした。これは私が踏んだ落とし穴です。
J君のような態度は、いわゆる`「中国の詰め込み教育」`の術中にはまっているのは明らかで、私は結局J君と一緒に堕落することはありませんでした。

高校二年でクラスが変わり、また別の同級生に出会いました。Yと呼びましょう。
Yは学習にとても勤勉で努力家で、規則正しい生徒でした。Yが毎日何時まで勉強しているのか、家に帰ってから勉強以外に何もしていないのか、そういったことは私には分かりません。しかし、学校で私が見かけるYは、ラジオ体操など学習に使えない時間帯を除いて、Yはいつも刻苦勉励、努力していました：授業中は真面目にノートを取り、拝読しなくても、彼が黒板の先生の一字一句を書き写しているだろうと推測できました。誤字も含めてかもしれません。休み時間は真面目に宿題を書き、分からない問題があると——その頻度はほぼ一問ごとに一度（やったことのある問題を除く）——クラスで成績の良い同級生や先生に質問しに行きます。彼らが誰かと話していたり、簡単には中断すべきでない作業をしているかどうかはまったく気にしません。
Yは授業中によく、先生が説明したばかりのところについて質問しました：どういう意味か分からない、と。実際、その部分はたった今説明されたばかりで、その時Yは前の板書を写していたのです——一字一句漏らさず写していたら、先生の説明に追いつくことは永遠に不可能です。

私はよく、もし自分がYの半分でも努力していたら、成し遂げられないことは何もなかっただろうと感慨にふけりました。そしてY自身は、私の目には何一つ成し遂げていないように映りました。表面的には努力しているように見えますが、実は退屈で融通がき# 1. 两数之和

## 题目

给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出 **和为目标值** *`target`* 的那 **两个** 整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案。但是，数组中同一个元素在答案里不能重复出现。

你可以按任意顺序返回答案。

**示例 1：**

```
输入：nums = [2,7,11,15], target = 9
输出：[0,1]
解释：因为 nums[0] + nums[1] == 9 ，返回 [0, 1] 。
```

**示例 2：**

```
输入：nums = [3,2,4], target = 6
输出：[1,2]
```

**示例 3：**

```
输入：nums = [3,3], target = 6
输出：[0,1]
```

**提示：**

- `2 <= nums.length <= 104`
- `-109 <= nums[i] <= 109`
- `-109 <= target <= 109`
- **只会存在一个有效答案**

**进阶：**你可以想出一个时间复杂度小于 `O(n2)` 的算法吗？

## 思路

使用哈希表，遍历数组，将数组元素作为 key，索引作为 value 存入哈希表，在遍历过程中，判断 target - 当前元素是否在哈希表中，如果在，则返回当前索引和哈希表中对应元素的索引

## 代码

```ts
function twoSum(nums: number[], target: number): number[] {
  const map = new Map<number, number>();
  for (let i = 0; i < nums.length; i++) {
    const num = nums[i];
    const diff = target - num;
    if (map.has(diff)) {
      return [map.get(diff)!, i];
    }
    map.set(num, i);
  }
  return [];
}
```
