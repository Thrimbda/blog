**この記事は、私の消え去らぬ童心から生まれました。**

4月、ある旅の途中で、私は強烈な衝動に駆られて旅行記を書きました。まずは機核に投稿し、その後はいつものように、Markdown形式に整えて自分のブログに載せるつもりでした。

ところが編集を終えた後、ブログが壊れていることに気づいたのです。

## 以前のブログ

個人ブログを立ち上げるブームは、大学3年生の時に私の学科で流行りました。7年前に書いた[この記事](<[搭好博客后的一点话](./da-hao-bo-ke-hou-de-yi-dian-hua/)>)（今読むと恥ずかしさで足の指が丸まるような内容です）が、私が初めてこのブログを構築した時の心境を紹介しています。

その理由は、まず自分に何かを書くことを促したかったから。次に、学校では教えてくれない、最新で流行りの華やかな技術を組み合わせたエンジニアリングの実践をしたかったから。そして、独自ドメインを持つのは本当にクールだと思ったからです。少なくとも私はそう感じていました。

当時の様子を再現してみましょう。

---

何か新しいことを少し学ぶたびに、興奮しながら、心に浮かんだアイデアをMarkdown形式の文書として記録しました。

書き終えたら、さっそうと数行のコマンドを実行して新しいブログをプッシュします。

```shell
$ git add .
$ git commit -m "new blog! hahahaha"
$ git push origin master
```

それからワクワクしながらCIの画面を開きます。当時CIはまだ比較的新しい概念で、DevOpsの理念が提唱されてから数年しか経っていませんでした。CIのログページで、この記事を含むブログ全体が静的なhtmlサイトとして構築されるのを待ち、Travisが私の鍵を使ってビルド成果物をgitリポジトリにプッシュするのを待ち、Githubがビルド成果物を公開するのを待ちました。

最後に私のドメイン [https://blog.thrimbda.com](https://blog.thrimbda.com) にアクセスし、ページの一番上に新しい記事が表示されているのを満足げに見つめました。

---

年月が経つにつれ、このプロセスは退屈なものになりました。退屈でありながらも継続的に機能し続け、ついにそれが機能しなくなる日が訪れるまで。

## 新しいブログ

4月末のある日、CIパイプラインを3回連続で再実行しても全て失敗しました。ログをざっと見ると、ビルドに使用するある依存関係が古すぎて、インターネット上から消えてしまったようでした。私は、この業界において7年という時間がいかに巨大なスパンであるかを思い知らされました。砂嵐が城を壊し、海水が山を飲み込むほどに。

そこで、仕事の合間の「サボり時間」に、あまり面白くはない要件をもとに技術選定を行いました：**十分にシンプルで、十分に便利なもの。**

こうして新しいブログが誕生しました。お決まりのように紹介します：

- ジェネレータとして [Zola](https://www.getzola.org/) を選択。
- 非常にシンプルですがなかなか良い見た目の[テーマ](https://github.com/Speyll/anemone)を物色。

旧ブログの代替として、この新しいブログは完成しました。

## 童心をくすぐるいじくり回しの過程

おそらく、十分に新しいから、あるいは技術選定が十分にシンプルだから、あるいはフロントエンド作業のフィードバックループが十分に短いからかもしれません。それに、阿錚と一緒に仕事をしていると、自然とフロントエンドの知識が耳に入ってきて、大学1年生の時にあまり真剣に学ばなかった知識の穴を埋めることができました。その結果、ブログをいじくり回す過程が再び面白くなってきたのです。

CSSスタイルのちょっとした修正、新しいgitcusの追加、コメント欄をgithubにホストする設定など、小さな試みをいくつか行いました。私は7年前の自分と意見が一致しました：コーディングとエンジニアリングのいじくり回しは、やはり楽しいことなのだと。それは労働の喜びであり、遊びの喜びです。

この喜びの余韻に乗じて、私が「機組」をブログに移行した技術的プロセスを（自己満足的に）詳しく説明したいと思います。

interface IGcoresTalk {
  text: string;
  images: string[];
  published_at: number;
  tags: string[];
}

const rawGcoresTalkData$: Observable<any[]> = of({
  before: Date.now() / 1000,
}).pipe(
  // 生データ
  expand(async ({ before }) => {
    url.searchParams.set("before", `${before}`);
    const res = await fetch(url);
    const data = await res.json();

      return EMPTY;

    return {

  takeWhile((v) => !!v.data),
  // filter((v) => v.data.length > 0),

    console.info(v);

  toArray(),
  shareReplay(1)
);

#### データ処理

生データが手に入ったので、次はMarkdownに処理します。直接処理を始めましょう。ポイントは以下の通りです：

- 表示する内容を決める
- 機核のインターフェースフィールドを理解する

機組は、友達の輪や微博などと似ていて、文字と画像、それにいくつかのタグで構成されています。そのまま平たく表示することにしましょう。
