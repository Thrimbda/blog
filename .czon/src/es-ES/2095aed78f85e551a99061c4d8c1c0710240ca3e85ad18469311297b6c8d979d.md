# /admin 使用说明（MVP）

## Objetivo

Sin necesidad de introducir un backend propio:

- Editar cualquier archivo `content/**.md`
- Los borradores se guardan automáticamente en el equipo local (IndexedDB)
- Al publicar, el contenido Markdown y las imágenes se escriben de vuelta en el repositorio de GitHub (por defecto: crear una rama + PR)

## Configuración

1. Copiar `static/admin/config.example.js` como `static/admin/config.js`
2. Crear una GitHub OAuth App y completar `oauthClientId`
   - Homepage URL: `https://0xc1.space/`
   - Callback URL: `https://0xc1.space/admin/` (no se usa para device flow, pero GitHub lo requiere en el formulario)

> `config.js` no debe contener ningún secreto; aquí solo se necesita `client_id`.
>
> Si deseas que **el sitio publicado también pueda usar `/admin/` directamente**, incluye `static/admin/config.js` en el repositorio (no contiene secretos, es seguro hacerlo público).

## Uso

1. Abrir `/admin/`
2. Hacer clic en “Connect (Device Flow)”
3. Seguir las instrucciones en la página para ingresar el código en GitHub y completar la autorización
4. Actualizar la lista de archivos para cargar `content/**.md`
5. Editar el contenido (los borradores se guardan automáticamente en el equipo local)
6. Insertar imágenes (se escribirán en `static/images/uploads/...` al publicar)
7. Hacer clic en “Publish (PR)” para crear una PR; al fusionarla, se activará el flujo de publicación del sitio
