# Notion MCP-Einrichtung für OpenCode

## Konfiguration abgeschlossen

Notion MCP wurde erfolgreich über den Remote-MCP-Server von OpenCode konfiguriert. Die Konfiguration ist in `opencode.json` gespeichert:

```json
{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "notion": {
      "type": "remote",
      "url": "https://mcp.notion.com/mcp",
      "enabled": true
    }
  }
}
```

## Authentifizierungsstatus

✅ **Erfolgreich authentifiziert**

Ihr Notion-Konto wurde über OAuth verbunden. Authentifizierungstoken werden sicher im lokalen Speicher von OpenCode aufbewahrt.

## Verwendung

### 1. MCP-Serverstatus überprüfen
```bash
opencode mcp list
```

### 2. Notion-Tools in OpenCode verwenden
Bei der Nutzung von OpenCode können Sie in Ihren Prompts auf Notion-Tools verweisen. Beispiel-Prompts:

```
Durchsuche meine Notion-Arbeitsbereiche nach Seiten zum Thema "Projektplanung" mithilfe der Notion-Tools
```

```
Liste meine neuesten Notion-Datenbanken mithilfe der Notion-Tools auf
```

```
Rufe den Inhalt einer bestimmten Notion-Seite mithilfe der Notion-Tools ab
```

### 3. Verfügbare Notion-Tools
Der Notion MCP-Server stellt Tools bereit für:
- Durchsuchen von Seiten und Datenbanken
- Abrufen von Seiteninhalten
- Erstellen und Aktualisieren von Seiten
- Arbeiten mit Datenbanken (jetzt Datenquellen genannt)
- Verwalten von Kommentaren
- Und mehr...

### 4. Bei Bedarf erneut authentifizieren
Falls die Authentifizierung abläuft:
```bash
opencode mcp auth notion
```

### 5. Authentifizierung entfernen
```bash
opencode mcp logout notion
```

## Sicherheitshinweise

- OAuth-Token werden sicher in `~/.local/share/opencode/` gespeichert
- Der Remote-MCP-Server (https://mcp.notion.com/mcp) wird von Notion betrieben
- Der Zugriff auf Ihre Notion-Daten ist auf das beschränkt, was Sie während der OAuth-Autorisierung freigegeben haben
- Es werden keine lokalen API-Token benötigt oder gespeichert

## Fehlerbehebung

Falls die Tools nicht zu funktionieren scheinen:

1. Verbindungsstatus überprüfen:
   ```bash
   opencode mcp debug notion
   ```

2. Erneut authentifizieren:
   ```bash
   opencode mcp logout notion && opencode mcp auth notion
   ```

3. OpenCode-Protokolle auf Fehler überprüfen

## Alternative: Lokaler MCP-Server

Falls Sie den lokalen MCP-Server mit Ihrem API-Token anstelle von OAuth bevorzugen:

1. `opencode.json` aktualisieren:
```json
{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "notion": {
      "type": "local",
      "command": ["npx", "-y", "@notionhq/notion-mcp-server"],
      "environment": {
        "NOTION_TOKEN": "{env:NOTION_TOKEN}"
      }
    }
  }
}
```

2. `.env`-Datei aktualisieren:
```
NOTION_TOKEN=hier_ihr_notion_integration_token_einfügen
```

3. OpenCode neu starten
