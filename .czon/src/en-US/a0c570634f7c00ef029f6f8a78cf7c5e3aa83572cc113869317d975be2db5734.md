## Detailed Design Plan

### Core Workflow
1. Modify the image output logic of `get-gcores-talk.ts`: Change the `slideshow` macro to multiple independent markdown image lines, each as `![...](...)`.
2. Modify `templates/shortcodes/slideshow.html`: Output an Embla-compatible HTML structure (`.embla`, `.embla__container`, `.embla__slide`), while retaining the outer `.slider-container` wrapper for style compatibility.
3. Add Embla and Embla Autoplay CDN script tags to `templates/head.html`.
4. Add Embla initialization code to `static/js/script.js` (or create a new script) to automatically initialize `.embla` containers, skipping those already initialized.
5. Add CZON's Embla CSS styles to `static/css/style.css`, or embed them via a style tag.
6. Remove the existing slider's JavaScript code from `static/js/script.js` (or keep it but disable it).

### Interface Definition
- Embla container structure:
```html
<div class="slider-container">
  <div class="embla">
    <div class="embla__container">
      <div class="embla__slide"><img src="..." /></div>
      ...
    </div>
  </div>
</div>
```

### File Change Details
- `scripts/get-gcores-talk.ts`: Modify image output logic.
- `templates/shortcodes/slideshow.html`: Rewrite to Embla structure.
- `templates/head.html`: Add Embla script tags.
- `static/js/script.js`: Add Embla initialization function; remove existing slider code.
- `static/css/style.css`: Add Embla styles; adjust existing slider styles.

### Uncertainties
- Does the existing slider's CSS need a complete rewrite? The outer `.slider-container` styles can be kept, with new styles applied to the internal Embla structure.
- Could Embla's auto-grouping logic unintentionally affect images on other pages? Need to ensure initialization only targets image groups containing the `.embla` class or specific containers.
