# /admin-Bedienungsanleitung (MVP)

## Ziel

Ohne Einführung eines eigenen Backends:

- Bearbeiten beliebiger `content/**.md`-Dateien
- Automatische Speicherung von Entwürfen auf dem lokalen Gerät (IndexedDB)
- Beim Veröffentlichen werden Markdown + Bilder in das GitHub-Repository zurückgeschrieben (Standard: Erstellen eines Branches + PR)

## Konfiguration

1. Kopieren Sie `static/admin/config.example.js` zu `static/admin/config.js`
2. Erstellen Sie eine GitHub OAuth App und tragen Sie die `oauthClientId` ein
   - Homepage-URL: `https://0xc1.space/`
   - Callback-URL: `https://0xc1.space/admin/` (wird für den Device Flow nicht benötigt, aber das GitHub-Formular erfordert sie)

> `config.js` sollte keine Secrets enthalten; hier wird nur die `client_id` benötigt.
>
> Wenn Sie möchten, dass **die veröffentlichte Website `/admin/` direkt nutzen kann**, geben Sie `static/admin/config.js` mit in das Repository ein (es enthält keine Secrets und kann problemlos öffentlich sein).

## Verwendung

1. Öffnen Sie `/admin/`
2. Klicken Sie auf "Connect (Device Flow)"
3. Folgen Sie den Anweisungen auf der Seite, um den Code auf GitHub einzugeben und die Autorisierung abzuschließen
4. Aktualisieren Sie die Dateiliste, um `content/**.md` zu laden
5. Bearbeiten Sie Inhalte (Entwürfe werden automatisch lokal gespeichert)
6. Fügen Sie Bilder ein (werden beim Veröffentlichen in `static/images/uploads/...` geschrieben)
7. Klicken Sie auf "Publish (PR)", um einen PR zu erstellen; nach dem Mergen wird der Website-Veröffentlichungsprozess ausgelöst
