## MVP (Vorgeschlagener Umfang)

- [ ] `/admin`-Seite: Wiederverwendung von `reset.css`/`suCSS.css`/`style.css` und Anbindung des bestehenden Light/Dark-Toggles (Stilkonsistenz beibehalten).
- [ ] Supabase: Erstellung der `documents`-Tabelle + RLS (Allowlist für Einzelbenutzer).
- [ ] Editor: Integration von Vditor (Priorität auf IR/Quelltext-Modus als Fallback), Implementierung von "Öffnen → Bearbeiten → Automatisches Speichern (debounce)".
- [ ] Front matter-Schutz: Frontend speichert/bearbeitet Front matter und Haupttext getrennt (Vermeidung von WYSIWYG-Überschreibungen).
- [ ] Veröffentlichungs-Button: Schreibt `published_at` (oder Boolean) zur Unterscheidung von "Entwurf/Veröffentlicht".
- [ ] GitHub Actions: Holt `published`-Dokumente aus Supabase, schreibt sie synchron in `content/**`, committet in `master` und löst Site-Deployment aus.
- [ ] Konfliktstrategie: Supabase standardmäßig als wahre Quelle; bei manuellen Repo-Änderungen Fail-Fast mit Hinweis auf manuelle Bearbeitung.

## v2 (Erweiterungen)

- [ ] Bild-Upload: Anbindung an Supabase Storage (Drag & Drop/Einfügen-Upload), automatisches Einfügen von Markdown-Links.
- [ ] Versionsverlauf: `document_revisions`-Tabelle (Diff/Rollback) oder Beibehaltung von Rollback-Punkten in Git-Commits.
- [ ] Konsistente lokale Vorschau: Bereitstellung eines "gemäß Zola-Renderregeln"-Vorschau-Hinweises in `/admin` (mindestens: Code-Highlighting, mermaid, Shortcodes unverändert anzeigen).
- [ ] Multi-Dokumenten-Management: Baumartiges Durchsuchen/Suchen/Taggen gemäß `content/**`-Struktur.
- [ ] Berechtigungserweiterungen: Mehrbenutzer (Mitarbeiter) und Audit-Logs.
- [ ] Veröffentlichungs-Workflow-Erweiterung: Generierung eines PR anstelle eines direkten Pushs, zur einfacheren Review.
