# Instrucciones de uso de /admin (MVP)

## Objetivo

Sin necesidad de introducir un backend propio:

- Editar cualquier archivo `content/**.md`
- Guardar borradores automáticamente en la máquina local (IndexedDB)
- Al publicar, escribir el Markdown y las imágenes de vuelta en el repositorio de GitHub (por defecto: crear rama + PR)

## Configuración

1. Copiar `static/admin/config.example.js` como `static/admin/config.js`
2. Crear una GitHub OAuth App y rellenar `oauthClientId`
   - Homepage URL: `https://0xc1.space/`
   - Callback URL: `https://0xc1.space/admin/` (el device flow no lo usa, pero el formulario de GitHub lo requiere)

> `config.js` no debe contener ningún secreto; aquí solo se necesita el `client_id`.
>
> Si deseas que **el sitio publicado también pueda usar `/admin/` directamente**, debes incluir `static/admin/config.js` en el repositorio (no contiene secretos, es seguro hacerlo público).

## Uso

1. Abrir `/admin/`
2. Hacer clic en "Connect (Device Flow)"
3. Seguir las instrucciones de la página para introducir el código en GitHub y completar la autorización
4. Actualizar la lista de archivos para cargar `content/**.md`
5. Editar el contenido (se guarda automáticamente como borrador en local)
6. Insertar imágenes (se escribirán en `static/images/uploads/...` al publicar)
7. Hacer clic en "Publish (PR)" para crear un PR; al fusionarlo se activará el flujo de publicación del sitio
