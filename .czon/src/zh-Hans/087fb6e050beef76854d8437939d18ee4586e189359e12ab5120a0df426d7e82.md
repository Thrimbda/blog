大家好！距离我们上次见面已经过去好长一段时间了，你们过得怎么样？

我在字节跳动的新工作一直非常忙碌，那为什么这篇小博客要用英文写呢？我会解释的。

<!--more-->

## 记录、反思与工具

我一直有强烈的冲动去记录自己的一天，看看自己在做什么，并对此进行统计。一段时间后，我发现了两个要点：记录一天并进行基于此的反思非常非常有帮助——我在六月份的那些反思最终帮助我做出了换工作的决定。然而，对于细粒度的时间追踪，我却一直失败。

也许问题出在我的工具上。我尝试过 Excel、自定义的 JSON 格式、用于自动时间追踪的 Time Rescue 应用（结果发现我从未真正反思过，因为信息太杂了——它几乎记录了你电脑上的所有事情，而像我这样的人并不真的需要这个……）、手机上的几个应用……我几乎要放弃了。

但我还是设法养成了记录每日笔记的习惯。有一天，我正在说服一位好朋友做笔记有多么重要，他对我使用的工具提出了质疑，那是我第一次认真考虑 `emacs` 和 `org-mode`。

他说：“伙计，我们说的是软件！有什么工具能存活个5年吗？”

我说：“哦，那样的话，也许你可以考虑一下 `org-mode`。”

我知道他是在开玩笑地挑剔我的推荐，但我突然想到：“何不试试看呢？”

## Spacemacs 与 Org Mode

我使用 Vim 已经两年了，我热爱 Vim 并曾为之狂热过一段时间（我总是遇到一些东西就觉得它是银弹，比如 Vim，比如 Scala，等等）。现在我知道，它只是一个设法在过去30年里存活下来的工具。它的很多设计是由于历史原因，而不是因为它本应如此（顺带一提，如果你对[原因](https://www.youtube.com/watch?v=0M6erlK57go&ab_channel=MikeZamansky)感兴趣，这个视频很棒，从历史角度告诉你为什么 emacs 和 vim 是这样设计的）。

那么，当一个 Vim 用户考虑来自 Emacs 世界的东西时，他会怎么做呢？很自然地，会想到像 Spacemacs 这样的东西：带有 Vim 键绑定的 Emacs。是的，它们可以和谐共存。

然后你大概能猜到我是如何开始使用 Spacemacs（这是我的[配置](https://github.com/Thrimbda/.thript)）和 Org Mode 的。我仍在学习，适应这个工具，并开始用 Org Mode 记录我的每日笔记。然而，有一个问题我还没解决：用 Vim 键绑定打中文很不自然，我还没找到一个好的解决方案。这就是为什么我用英文写这篇博客。而且，当你用另一种语言表达时，感觉会不一样，就像变成了另一个人，你可能永远看不到我用如此直白的风格嘀咕关于自己的事情。

不过，能和大家聊聊还是很开心的 :)

另外，我还没找到在我的 Hexo 博客中用 Org 替代 Markdown 的好方法……
