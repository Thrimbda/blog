# 设计系统：c1 博客

## 1. 概述

**创意北极星：「终端纸张存档」**

视觉系统将暖色调纸张颜色与类似终端的等宽节奏相结合。它有意保持安静：内容、时间顺序和简洁导航就是界面。设计应感觉是维护良好的、个性化的，而非打磨成通用的出版物产品。

当前站点摒弃了装饰性表面。规则、括号、ASCII标记、文本链接和狭窄的阅读列构成了身份标识。未来的工作应保留这种语言，并改进阅读连续性、响应适应性和状态清晰度，同时避免引入卡片密集型布局或营销页面设计。

**关键特征：**
- 文本优先，低装饰，高连续性。
- 浅色模式下的温暖纸张和深色模式下的棕黑色纸张。
- 通过自定义 `Dashes` 字体实现等宽衬线回退。
- 使用细规则和括号式控件，而非填充式小部件。
- 静态优先行为，JavaScript 作为渐进增强。

## 2. 色彩

调色板是一个温暖的存档纸张系统，配有一个克制的蓝色链接强调色和紫色已访问状态。

### 主色
- **链接蓝色**（`#1f4d8f`）：在浅色模式中唯一强烈的交互强调色。用于行内链接和重要的导航提示。
- **已访问紫色**（`#6a537d`）：保留类似浏览器的已访问语义，而不显得装饰性过强。

### 中性色
- **纸张背景**（`#f6f1e8`）：默认页面背景和主导表面。
- **凸起纸张**（`#fbf8f2`）：下拉菜单和小型凸起 UI 表面。
- **墨水**（`#171411`）：正文和主要标题。
- **柔和墨水**（`#6c655c`）：辅助元数据、列表标记和低强调提示。
- **强柔和墨水**（`#555047`）：导航、控件和需要保持可读性的元数据。
- **分隔线**（`#d4ccbf`）：分隔线、边框、焦点轮廓和文章列表引导线。
- **深色纸张**（`#171311`）：深色模式页面背景。
- **深色凸起纸张**（`#221b17`）：深色模式凸起表面。
- **深色墨水**（`#efe2d2`）：深色模式正文。
- **深色分隔线**（`#4a4038`）：深色模式分隔线和边框。

### 命名规则

**单一强调色规则。** 蓝色仅用于链接和核心交互提示。不要在长阅读页面上添加装饰性强调色。

**纸张保持纸张规则。** 保留温暖的纸张身份，但当柔和文本或控件放置在有色背景上时，务必检查对比度。

## 3. 排版

**显示字体：** `Dashes`，本地 Georgia、Times New Roman、DejaVu Serif、Liberation Serif、Nimbus Roman 作为回退字体。
**正文字体：** `Dashes`、`monospace`。
**标签/等宽字体：** 与正文相同的字体族。

**特征：** 字体风格在机械感和文学感之间故意平衡。它应该读起来像打印在纸张上的耐用文本控制台，而不是现代 SaaS 界面。

### 层次结构
- **显示**（700，`1.52em`，行高继承自正文）：页面标题和主要文章标题。
- **标题**（700，`1.24em`）：页面内和长文中的章节标题。
- **标题（次级）**（700，`1.1em`）：低级标题和紧凑章节标题。
- **正文**（400，`15px`，`1.72`）：长文阅读。保持文章宽度在 `69ch` 到 `70ch` 左右。
- **标签**（600，`0.8rem`，`0.08em` 字间距）：稀疏控件，如轮廓摘要。不要在章节间滥用标签样式。

### 命名规则

**阅读列规则。** 正文内容应保持在 `65ch` 到 `75ch` 之间；当日期/列表结构需要时，长日志索引可以更宽。

## 4. 层级

系统默认是扁平的。深度通过细规则、色调表面、间距和源代码顺序来表达。阴影很少见，且应是结构性的，而非装饰性的；语言切换下拉菜单是当前的例外，因为它浮动在标题流之上。

### 阴影词汇
- **下拉提升**（`box-shadow: 0 0.45rem 1rem rgba(23, 20, 17, 0.08)`）：仅用于需要在视觉上与页面分离的小型浮动控件。

### 命名规则

**扁平存档规则。** 文章、日志条目和列表没有卡片阴影。使用标题、日期、规则和空白来组织结构。

## 5. 组件

### 按钮
- **形状：** 方形文本控件，无圆角（`0`）。
- **主要：** 没有填充式主要按钮模式。使用文本链接或括号式幽灵按钮。
- **悬停 / 焦点：** 悬停/焦点时加下划线；`1px` 分隔线颜色的焦点轮廓，带有小偏移。
- **幽灵：** 主题切换使用括号伪元素和页面背景上的柔和文本。

### 标签
- **样式：** 标签是呈现为 `/tag/` 的文本链接，而非胶囊形状。
- **状态：** 目前没有已选标签样式；除非需要更强状态，否则保持未来筛选为文本形式。

### 卡片 / 容器
- **角样式：** 无卡片圆角。
- **背景：** 文章直接放置在页面背景上。
- **阴影策略：** 内容容器无阴影。
- **边框：** 细规则分隔页眉、页脚、大纲栏、代码块、表格和列表引导线。
- **内边距：** 内容依赖页面填充和行节奏，而非盒状内边距。

### 输入框 / 字段
- **样式：** 原生表单控件继承等宽字体族，并使用 `1px` 规则边框。
- **焦点：** 优先使用可见的分隔线颜色轮廓。
- **错误 / 禁用：** 目前尚未定义；未来的表单应保持状态文本优先、高对比度。

### 导航
- **样式：** 页眉和页脚导航是内联文本链接，位于 ASCII 标志下方。控件使用括号文本。导航在窄屏上折行，而非折叠为汉堡菜单。
- **移动端处理：** 正文内边距压缩，文章列表从引导线行切换为两列/日期在标题下的行，下拉菜单变为流内面板。

### 页面大纲

大纲栏是长文中的标志性组件。在桌面端，它成为带细分隔线的固定左侧栏；在较小屏幕上，它保留在文档流中。折叠状态应基于路径持久化，且绝不能隐藏文章本身。

### Embla 图片组

连续图片被逐步分组到 Embla 轮播中。轮播应保持图片可检查、受视口高度限制，且不应改变单张图片的文章。

## 6. 做与不做

### 应做：
- **应当** 在添加新 UI 时，保留 `--bg`、`--fg`、`--muted`、`--link` 和 `--border` 标记词汇。
- **应当** 将长文宽度保持在接近 `70ch`，列表/存档页面保持在现有 `84ch` 框架内。
- **应当** 在添加面板之前，优先使用细 `1px` 规则、括号式控件和文本链接。
- **应当** 为懒加载或无限浏览界面提供非 JS 回退。
- **应当** 在移动端、笔记本和宽屏桌面端测试溢出、触摸目标和行长。

### 不应做：
- **不应** 引入卡片密集型存档网格、光面渐变、玻璃面板或英雄区域。
- **不应** 添加超过 `1px` 的 `left-border` 或 `right-border` 强调条纹。
- **不应** 为普通存档导航使用大型圆角卡片、胶囊标签或填充式 CTA 按钮。
- **不应** 将日常日志条目隐藏在仅 JavaScript 渲染之后。
- **不应** 让导入的图片、长 URL、代码块或日记标题在移动端产生水平滚动。
