## 记录、反思与工具

我一直有强烈的冲动去记录自己的一天，看看自己在做什么，并对此进行统计。
一段时间后，我发现了两件事：记录我的一天并基于此进行反思非常非常有用，我在六月份反思的内容最终帮助我做出了换工作的决定。然而，对于细粒度的时间追踪，我一直都在失败。

也许问题出在我的工具上。我尝试过 Excel、自定义的 JSON 模式、用于自动时间追踪的 Time Rescue 应用（结果发现我从未真正反思过，因为噪音太多，它几乎记录了你电脑上的所有事情，而像我这样的人并不真的需要这个……）、手机上的几个应用……我几乎要放弃了。

但我设法养成了记录每日笔记的习惯。有一天，我正在说服我的一个好朋友做笔记有多么重要，他向我提出了关于工具的问题，那是我第一次认真考虑 `emacs` 和 `org-mode`。

他说：“伙计，我们说的是软件！有什么工具能存活个五年吗？”

我说：“哦，如果是那样的话，也许你可以考虑一下 `org-mode`。”

我知道他是在开玩笑，对我的推荐吹毛求疵，但我想：“为什么不试试呢？”

## Spacemacs 与 Org Mode

我使用 Vim 已经两年了，我热爱 Vim，并且对 Vim 狂热了很长一段时间（我总是遇到一些东西，就认为它是银弹，对于 Vim，对于 Scala 等等）。现在我知道，它只是一个设法在过去 30 年里存活下来的工具。它的很多设计是由于历史原因，而不是因为它本应如此（顺带一提，也许你对[为什么](https://www.youtube.com/watch?v=0M6erlK57go&ab_channel=MikeZamansky)感兴趣，这是一个非常棒的视频，从历史角度告诉你为什么 emacs 和 vim 是这样设计的）。
那么，当一个 Vim 用户考虑来自 emacs 世界的东西时，他会做什么呢？自然是像 Spacemacs 这样的东西：带有 Vim 键绑定的 emacs，是的，它们可以和谐共存。

然后你大概能猜到我是如何开始使用 Spacemacs（这是我的[配置](https://github.com/Thrimbda/.thript)）和 org mode 的。我仍在学习，适应这个工具，并开始使用 org mode 记录我的每日笔记。然而，有一个问题我无法解决：使用 Vim 键绑定输入中文很不自然，我还没有找到一个好的解决方案。这就是为什么我用英文写这篇博客。而且，当你用另一种语言说话时，感觉会不一样，感觉像是另一个人，你可能永远看不到我用如此直白的风格自言自语。

不过，能和大家聊聊还是很开心的 :)

另外，我还没有找到一个好的解决方案，在我的 hexo 博客中用 org 替代 markdown...
