# Product Context

## What This Is

This repository builds `0xc1.space`, a personal static blog and archive. It contains long-form writing, diary/work logs, imported Gcores fragments, multilingual content, and hand-maintained theme code.

## Register

`brand`

The design is part of the product. The site should feel like a durable personal archive rather than a generic dashboard or publishing SaaS.

## Audience

- The author, who needs a repairable static system for long-term writing and logging.
- Returning readers who browse posts, diary fragments, and small life/work records over time.
- Future agents or maintainers who need predictable file organization, templates, and migration rules.

## Product Principles

- Preserve archive durability over runtime cleverness.
- Keep long reading quiet, text-first, and scannable.
- Prefer static Zola pages and links before JavaScript enhancement.
- Keep old URLs understandable during migrations.
- Make daily fragments independently linkable without making browsing feel fragmented.

## Constraints

- Zola remains the static site generator.
- The current terminal-paper visual identity is intentional.
- JavaScript may improve browsing, but it must not be the only way to reach content.
- The archive includes duplicate Gcores dates, so date-only URLs are not always unique.
